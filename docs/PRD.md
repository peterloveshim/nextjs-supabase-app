# 모임 & 이벤트 관리 웹 MVP PRD

## 핵심 정보

**목적**: 소규모 지인 모임의 주최자가 공지, 참여자 관리, 카풀, 정산을 한 곳에서 처리할 수 없는 문제를 해결한다
**사용자**: 수영, 헬스, 친구 모임 등 10~30명 규모의 소규모 지인 모임을 운영하는 주최자와 링크를 통해 참여하는 참여자

---

## 사용자 여정

### 주최자 흐름

```
1. 로그인 페이지 / 회원가입 페이지
   (이메일 또는 소셜 로그인)
   ↓

2. 대시보드 페이지
   (내가 만든 이벤트 목록 확인)
   ↓ [이벤트 생성 버튼 클릭]

3. 이벤트 생성 페이지
   (제목, 설명, 장소, 날짜/시간, 정원 입력)
   ↓ [생성 완료]

4. 이벤트 상세 페이지 (공지 탭)
   ↓ [참여 링크 복사 후 공유]

5. 이벤트 상세 페이지 (참여자 탭)
   (참여 신청 승인 / 거절)
   ↓

6. 이벤트 상세 페이지 (공지 탭)
   (공지 작성)
   ↓

7. 이벤트 상세 페이지 (카풀 탭)
   (카풀 드라이버 등록 또는 탑승 신청)
   ↓

8. 이벤트 상세 페이지 (정산 탭)
   (총액 입력 → 1/N 계산 → 납부 여부 체크)
```

### 참여자 흐름

```
1. 이벤트 공유 링크 클릭
   ↓
   [비로그인 상태] → 로그인 페이지 / 회원가입 페이지 → 로그인 후 자동으로 참여 신청 처리
   [로그인 상태]   → 이벤트 상세 페이지로 이동 → 참여 신청

2. 이벤트 상세 페이지 (공지 탭)
   (승인 전: 공지 비공개 / 승인 후: 공지 열람 가능)
   ↓ [주최자 승인 대기]

3. 이벤트 상세 페이지 (공지 탭)
   (승인 완료 후 공지 확인)
   ↓

4. 이벤트 상세 페이지 (카풀 탭)
   (카풀 드라이버 등록 또는 탑승 신청)
   ↓

5. 이벤트 상세 페이지 (정산 탭)
   (본인 납부 여부 확인)
```

## 기능 명세

### 1. MVP 핵심 기능

| ID       | 기능명               | 설명                                                                             | MVP 필수 이유                               | 관련 페이지                                        |
| -------- | -------------------- | -------------------------------------------------------------------------------- | ------------------------------------------- | -------------------------------------------------- |
| **F001** | 이벤트 생성          | 제목, 설명, 장소, 날짜/시간, 정원을 입력하여 이벤트를 생성한다                   | 서비스의 핵심 시작점                        | 이벤트 생성 페이지                                 |
| **F002** | 이벤트 수정/삭제     | 이벤트 정보 수정 및 상태 변경(open/closed/cancelled), 삭제                       | 주최자의 운영 기본 기능                     | 이벤트 수정 페이지, 이벤트 상세 페이지             |
| **F003** | 대시보드 이벤트 목록 | 내가 만든 이벤트와 내가 참여하는 이벤트를 구분하여 목록으로 표시한다             | 전체 이벤트 파악을 위한 진입점              | 대시보드 페이지                                    |
| **F004** | 링크 공유 참여 신청  | 이벤트 공유 링크로 접근 시 로그인 후 자동 참여 신청 처리                         | 모임 초대의 핵심 진입 경로                  | 이벤트 상세 페이지, 로그인 페이지, 회원가입 페이지 |
| **F005** | 참여 신청 승인/거절  | 주최자가 참여 신청 목록을 확인하고 승인 또는 거절한다                            | 모임 구성원 관리의 핵심 기능                | 이벤트 상세 페이지 (참여자 탭)                     |
| **F006** | 공지 작성 및 조회    | 주최자가 공지를 작성하고 승인된 참여자만 공지를 열람할 수 있다                   | 모임 정보 전달의 핵심 채널                  | 이벤트 상세 페이지 (공지 탭), 공지 작성 페이지     |
| **F007** | 카풀 드라이버 등록   | 드라이버가 출발지, 좌석 수, 메모를 등록한다                                      | 소규모 모임의 필수 이동 조율 기능           | 이벤트 상세 페이지 (카풀 탭), 카풀 등록 페이지     |
| **F008** | 카풀 탑승 신청       | 잔여 좌석을 확인하고 탑승 신청을 한다                                            | 카풀 기능의 참여자 측 핵심 동작             | 이벤트 상세 페이지 (카풀 탭)                       |
| **F009** | 정산 관리            | 총액 입력 시 1/N 자동 계산, 참여자별 납부 여부 주최자가 체크                     | 모임 후 정산 처리의 핵심 기능               | 이벤트 상세 페이지 (정산 탭), 정산 생성 페이지     |
| **F010** | 웹 내 알림           | 새 공지, 참여 신청, 정산 생성 시 Supabase Realtime으로 실시간 알림 뱃지 업데이트 | 주요 상태 변화를 놓치지 않기 위한 필수 기능 | 알림 목록 페이지, 공통 헤더                        |

## 메뉴 구조

```
공통 헤더 (모든 페이지)
├── 서비스명 로고 (대시보드로 이동)
├── 알림 아이콘 (실시간 뱃지) - F010
└── 인증 영역
    ├── 비로그인: 로그인, 회원가입 - F011
    └── 로그인:  로그아웃 - F011

주요 내비게이션 (로그인 후)
├── 대시보드 - F003
│   ├── 내가 만든 이벤트 목록
│   └── 내가 참여하는 이벤트 목록
├── 이벤트 생성 - F001
└── 알림 목록 - F010

이벤트 상세 내비게이션 (탭)
├── 공지 탭 - F006
├── 참여자 탭 - F004, F005
├── 카풀 탭 - F007, F008
└── 정산 탭 - F009
```

## 페이지별 상세 기능

### 로그인 페이지 / 회원가입 페이지

> **구현 기능:** `F011`, `F004` | **인증:** 비로그인 사용자 전용 (로그인 시 대시보드로 리디렉션)

| 항목            | 내용                                                                                                                                                                                                 |
| --------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **역할**        | 사용자 인증 처리 및 이벤트 공유 링크 접근 시 자동 참여 신청 연계                                                                                                                                     |
| **진입 경로**   | 직접 접근, 또는 이벤트 공유 링크 클릭 후 비로그인 상태로 리디렉션                                                                                                                                    |
| **사용자 행동** | 이메일/비밀번호로 로그인 또는 회원가입 처리                                                                                                                                                          |
| **주요 기능**   | 이메일/비밀번호 입력 폼 (React Hook Form + Zod 검증)<br>회원가입 시 이메일 인증 처리<br>공유 링크 경유 시 로그인 완료 후 해당 이벤트로 자동 리디렉션 및 참여 신청 처리<br>**로그인 / 회원가입** 버튼 |
| **다음 이동**   | 일반 로그인 성공 → 대시보드 페이지, 공유 링크 경유 로그인 성공 → 이벤트 상세 페이지 (자동 참여 신청), 실패 → 에러 메시지 표시                                                                        |

| **진입 경로**   | 이벤트 상세 페이지 카풀 탭에서 카풀 등록 버튼 클릭                                                                       |
| **사용자 행동** | 출발지, 좌석 수, 메모를 입력하고 카풀을 등록한다                                                                         |
| **주요 기능**   | 출발지 입력 (필수)<br>좌석 수 입력 (필수, 숫자)<br>메모 입력 (선택)<br>Zod 스키마 기반 유효성 검사<br>**카풀 등록** 버튼 |
| **다음 이동**   | 등록 성공 → 이벤트 상세 페이지 (카풀 탭), 취소 → 이벤트 상세 페이지 (카풀 탭)                                            |

> **구현 기능:** `F009` | **인증:** 로그인 필수 (주최자만 접근)

| 항목            | 내용                                                                                                                                                     |
| --------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- |

| **진입 경로**   | 이벤트 상세 페이지 정산 탭에서 정산 생성 버튼 클릭                                                                                                       |
| **사용자 행동** | 총액과 설명을 입력하면 승인된 참여자 수 기준으로 1인 금액이 자동 계산된다                                                                                |
| **주요 기능**   | 총액 입력 (필수, 숫자)<br>설명 입력 (선택)<br>1인 금액 자동 계산 미리보기 (총액 / 승인된 참여자 수)<br>Zod 스키마 기반 유효성 검사<br>**정산 생성** 버튼 |
| **다음 이동**   | 생성 성공 → 이벤트 상세 페이지 (정산 탭), 취소 → 이벤트 상세 페이지 (정산 탭)                                                                            |

## 기술 스택

### 프론트엔드 프레임워크

- **Next.js 16** (App Router) - React 풀스택 프레임워크, Fluid compute 패턴 적용
- **TypeScript 5.6+** - 타입 안전성 보장 (any 타입 사용 금지)
- **React 19** - UI 라이브러리 (동시성 기능 활용)

### 스타일링 & UI

- **TailwindCSS v4** - 설정 파일 없는 새로운 CSS 엔진, 모바일 우선 반응형 디자인
- **shadcn/ui (new-york 스타일)** - 고품질 React 컴포넌트 라이브러리
- **Lucide React** - 아이콘 라이브러리
- **next-themes** - 다크/라이트 테마 전환

### 폼 & 검증

- **React Hook Form 7.x** - 폼 상태 관리
- **Zod** - 스키마 검증 (폼 유효성 + API 응답 검증)

### 상태 관리 & 데이터 패칭

- **TanStack Query v5** - 서버 상태 관리 및 캐싱 (이벤트/공지/카풀/정산 목록)
- **Zustand** - 클라이언트 전역 상태 관리
- **nuqs** - URL 쿼리 파라미터 상태 관리 (이벤트 상세 탭 상태)

### 백엔드 & 데이터베이스

- **Supabase** - BaaS (인증, PostgreSQL, Realtime, RLS)
  - 인증: 이메일/비밀번호 기반 (`@supabase/ssr`)
  - Realtime: 알림 뱃지 실시간 업데이트 (notifications 테이블 구독)
  - RLS: 역할별 데이터 접근 제어 (주최자/승인된 참여자/일반 참여자)
- **PostgreSQL** - 관계형 데이터베이스 (Supabase 포함)

### 배포 & 호스팅

- **Vercel** - Next.js 16 최적화 배포 플랫폼

### 패키지 관리

- **npm** - 의존성 관리

| 테이블             | 읽기                  | 쓰기                  |
| ------------------ | --------------------- | --------------------- |
| events             | 모든 로그인 사용자    | 주최자(host_id)       |
| event_members      | 주최자, 본인 레코드   | 본인                  |
| announcements      | 주최자, 승인된 참여자 | 주최자                |
| carpools           | 주최자, 승인된 참여자 | 주최자, 승인된 참여자 |
| carpool_members    | 주최자, 승인된 참여자 | 본인                  |
| settlements        | 주최자, 승인된 참여자 | 주최자                |
| settlement_members | 주최자, 본인 레코드   | 주최자                |
| notifications      | 본인 레코드만         | 시스템(트리거)        |

### 역할별 UI 분기

- 이벤트 상세 페이지에서 **주최자**와 **참여자** UI를 명확히 분기
- 주최자 전용: 수정 버튼, 참여 승인/거절, 공지 작성, 납부 체크, 정산 생성
- 참여자 전용: 참여 신청 버튼 (미승인 상태일 때)
- 공통: 공지 열람(승인 후), 카풀 목록, 탑승 신청, 정산 금액 확인

## 섹션별 상세 내용

### Problem — 해결하는 문제

1. **분산된 모임 운영 도구**
   모임 주최자가 공지(카톡), 참여 신청(네이버 폼), 카풀(단체채팅), 정산(계좌이체 + 스프레드시트)을 각기 다른 곳에서 처리해야 한다.

2. **카풀 조율의 비효율**
   드라이버와 탑승자 매칭이 단체 채팅방에서 산발적으로 이루어져 누락·혼선이 잦다.

3. **정산 처리의 번거로움**
   엑셀·메모장으로 정산 내역을 관리하며, 납부 여부를 일일이 확인해야 한다.

---

### Customer Segments — 고객

| 구분                   | 설명                                          | 규모         |
| ---------------------- | --------------------------------------------- | ------------ |
| **주 고객** (주최자)   | 수영·헬스·등산·친구 정기 모임을 운영하는 개인 | 10~30명 모임 |
| **보조 고객** (참여자) | 링크로 초대받아 모임에 참여하는 구성원        | 주최자당 N명 |

**얼리어답터:** 정기적으로 동일 멤버와 활동 모임을 진행하는 2030세대 주최자

---

### Unique Value Proposition — 핵심 가치

> **"링크 하나로 모임을 만들고, 관리하고, 정산까지 완료"**

- 별도 앱 설치 없이 웹에서 즉시 사용
- 참여 신청부터 정산까지 컨텍스트 이동 없이 한 페이지에서 처리
- 링크 공유만으로 자연스러운 참여 유도

---

### Solution — 솔루션

| 문제      | 솔루션                                            |
| --------- | ------------------------------------------------- |
| 분산 도구 | 이벤트 생성 → 링크 공유 → 참여 신청 → 승인 원스톱 |
| 공지 전달 | 이벤트 상세 공지 탭 (승인된 참여자만 열람)        |
| 카풀 조율 | 드라이버 등록 + 잔여 좌석 표시 목록               |
| 정산 처리 | 총액 입력 → 1/N 자동 계산 → 납부 체크             |

---

### Channels — 채널

1. **이벤트 공유 링크** (핵심 바이럴 채널)
   주최자가 카카오톡·인스타그램·SMS로 링크 공유 → 참여자 유입 → 자동 가입

2. **지인 추천**
   참여자가 다음 모임의 주최자로 전환되는 자연 성장

3. **직접 가입**
   검색·입소문을 통한 신규 주최자 유입

---

| 지표                    | 의미                                    |
| ----------------------- | --------------------------------------- |
| 월간 활성 이벤트 수     | 서비스 핵심 활동 지표                   |
| 이벤트당 평균 참여자 수 | 모임 규모 및 바이럴 효과                |
| 주최자 재사용률         | 서비스 가치 검증 (2회 이상 이벤트 생성) |

| 링크 → 가입 전환율      | 바이럴 채널 효율                        |

---

### Unfair Advantage — 경쟁 우위

1. **초대 링크 = 바이럴 채널**
   모든 참여자가 링크를 통해 자연스럽게 신규 사용자로 전환됨. 별도 마케팅 없이 모임마다 신규 가입자 유입.

2. **Zero Install, Zero Config**
   앱 설치·복잡한 설정 없이 링크 클릭만으로 즉시 사용 가능.

3. **모임 맥락 통합**
   정산·카풀이 이벤트 안에 통합되어 있어, 도구 전환 없이 모든 모임 운영이 가능.
